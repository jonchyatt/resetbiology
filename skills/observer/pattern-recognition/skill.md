# Pattern Recognition Skill

## Purpose
Watches codebase and identifies patterns, anti-patterns, and opportunities for improvement.

## When to Use
- Continuous monitoring
- Code review
- Refactoring planning
- Performance optimization

## Operations

### identify_repeated_code
Finds code duplication
Parameters: directory

### analyze_performance
Identifies performance bottlenecks
Parameters: profiling_data

### suggest_refactoring
Suggests refactoring opportunities
Parameters: code_analysis

## Usage Examples
- "Use pattern-recognition to find repeated code in components"
- "Use pattern-recognition to identify performance issues"
